# Thai / English language system

Trail Runner Coach supports Thai (`th`) and English (`en`) without changing the user's health or training records.

## User experience

- The header button shows `EN` while Thai is active and `ไทย` while English is active.
- Language can also be selected in **Log > Settings**.
- The selection is stored in IndexedDB under `settings.language` and is restored on the next launch.
- Navigation, pages, forms, status messages, training guidance, rehab cues, nutrition guidance, gear, connection setup, and dialogs are localized.
- Dates and number formatting follow the active language.
- Food records use `nameTh` and `nameEn`. English mode displays `nameEn` when available and safely falls back to Thai.
- User-entered notes and motivation text remain in the language entered by the user.

## Architecture

- `public/js/core/i18n.js` owns supported languages, translations, patterns, DOM localization, and localized data helpers.
- `applyShellLanguage()` handles persistent shell elements such as bottom navigation and the language toggle.
- `localizeDom()` is a compatibility layer for existing view templates. It translates complete strings only, preventing mixed Thai/English labels.
- `app.t()`, `app.name()`, and `app.field()` should be used for newly developed dynamic content.

## Adding a new UI string

1. Prefer structured bilingual data for long-lived records:

```js
{ title: { th: 'วิ่งเบา', en: 'Easy run' } }
```

2. For a static UI label, add the Thai source and English value to `EN_EXACT` in `core/i18n.js`.
3. For a dynamic whole sentence, add an anchored regular expression to `EN_PATTERNS`.
4. Do not use partial word replacement. It can produce mixed-language sentences.
5. Run `npm test` and the route/tab bilingual coverage checks before release.

## Data behavior

Changing language does not migrate or duplicate workouts, food logs, Apple Health data, pain logs, or race profiles. It only changes presentation. The database name and schema version remain unchanged in v1.4.0.
